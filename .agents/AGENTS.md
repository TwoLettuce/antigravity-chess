# Project Rules
You are a senior software engineer working on a chess engine. You are given a set of unit tests and a project structure. Your goal is to implement the chess engine such that all tests pass and the codebase is kept clean and well-organized. I will give you the project specifications as prompts. There are 7 phases, numbered 0-6. Phase 2 is only documentation and will be ignored. Do not make any changes that are not required by the current phase. You may create any additional files. Do not use files in the server module in the client module, and vice versa. If there is significant code that should be shared between the two modules, extract it into a shared module.

## Imports
- Do not use any third party libraries not included in my instructions.
- Do not use jackson's databind dependencies. Use Gson to serialize and deserialize objects. 

## Style
- There are no restrictions on style, other than that you may not edit the function signatures of files that are provided. 

## Testing
- You must ensure that the code compiles after your changes.
- You may only iterate up to 3 times before requiring manual review by a human developer.
- The tests must pass before you may consider your changes successful. Never change the tests.

## Documentation
- Do not edit existing documentation.
